## Asset status

**Live-generate mode requested. Outcome: fallback to Picsum.**

The Gemini Nano Banana model (`gemini-3.1-flash-image-preview`) returned `429
RESOURCE_EXHAUSTED — Quota exceeded for metric:
generativelanguage.googleapis.com/generate_content_free_tier_input_token_count`
on the very first call. The provided `GEMINI_API_KEY` is on a free tier with
0 daily quota for the image-gen endpoint, so no image generation was possible
in this session.

Per the kodagen `ASSET MODE` override:
- Gradient `<div role="img">` placeholders are forbidden — every image slot
  uses a real image.
- The approved fallback chain (Picsum seeded URLs) is in use for every slot:
  hero, service cards, section parallax, contact photo, about portrait,
  resources thumbnails.

To upgrade to real Nano Banana keyframes + Veo scroll-scrub hero:

```bash

npm run gen:all
# Then wire the extracted frames into the Hero component (see prompts/scene-1/).
```

All prompts are pre-written in `prompts/scene-1/{start,end,motion}.txt`,

`prompts/section-{process,about}.txt`. The generation script (`scripts/generate-images.ts`)

## Build status

**Build is currently failing on the Next.js internal `/_global-error`
prerender.** This is a known Next 16 + framer-motion / motion 12
incompatibility with React 19's strict context handling. The application
**runs correctly in `next dev`** — the failure is isolated to the static
generation of the framework's auto-generated error fallback page, not any
user-authored page.

Pages affected: none in the user-facing site. All public routes
(`/`, `/about`, `/services/*`, `/resources`, `/consultation`, `/contact`)
are written with `dynamic = "force-dynamic"` inherited from the root layout,
so they render at request time without prerender.

Workarounds attempted in this session (none resolved the internal page):
1. Custom `app/global-error.tsx` override
2. `app/not-found.tsx` override
3. `dynamic = "force-dynamic"` on layout AND global-error
4. `framer-motion@12` → `motion/react` import migration
5. Downgrade React 19 → 18.3 and Next 16 → 15.5

The build can be unblocked by either:
- Pinning to Next 14.x with React 18 + framer-motion 11, OR
- Removing all framer-motion / motion imports and replacing entrance
  animations with Tailwind-driven CSS animations (no runtime hooks)
